Atlas-ASX Deep Research Report: Parameter Robustness & ML Enhancements for Systematic Trading

Date: 19 February 2026 | Prepared for: Atlas-ASX Trading System | Classification: Internal Research

Executive Summary

This report addresses four critical problems in the Atlas-ASX systematic trading system operating on ~185 ASX mid-cap tickers with $5K capital:

  1. Parameter Fragility (P1): +/-15% perturbation drops CAGR from 11.2% to 2.67% mean (76% degradation), with 2/10 trials going negative
  2. In-Sample/OOS Gap (P2): IS CAGR 17.09% drops to OOS 2.20% (87% degradation), OOS profit factor < 1.0
  3. Walk-Forward Inconsistency (P3): Only 59% (13/22) windows profitable
  4. Limited Trade Count (P4): ~287 trades over 2 years constrains statistical validation

Key Findings: The system suffers from classic backtest overfitting. The most impactful interventions are:

  1. Parameter plateau detection using PSO-based plateau scoring (directly fixes P1)
  2. Objective function smoothing over parameter neighborhoods (fixes P1, helps P2)
  3. Combinatorial Purged Cross-Validation (CPCV) replacing standard walk-forward (fixes P2, P3)
  4. Deflated Sharpe Ratio for honest strategy evaluation (fixes P2, P4)
  5. Meta-labeling to filter when to take signals (fixes P2, P3)

ML enhancements should be pursued cautiously given the small sample size (~287 trades). Meta-labeling and simple gradient boosting signal combination are the most practical options. Deep learning (TFT, attention) is NOT recommended at this scale.

PART 1: Parameter Robustness & Anti-Overfitting

1.1 Parameter Plateau Detection (PSO-Based)

CRITICAL Medium Complexity

What

Parameter plateau detection identifies broad, stable regions in parameter space where performance remains consistent under small perturbations, as opposed to sharp "parameter islands" that indicate overfitting. Wu et al. (2024) developed a plateau score algorithm combined with Particle Swarm Optimization (PSO) for efficient search in high-dimensional parameter spaces.

A parameter plateau is a continuous block of parameter combinations with performance comparable to nearby ranges. The plateau score quantifies robustness: higher scores = broader plateau = more robust. Traditional optimization finds the single best point (sharp peak); plateau-seeking finds the best region.

Three approaches exist:

  1. PSO with Plateau Score (Wu et al. 2024): For each candidate, evaluate neighbors in a defined radius; compute plateau score = fraction of neighbors meeting threshold x mean neighbor performance / base
  2. Heatmap Visualization: For 2-3 param dimensions, plot performance heatmaps to visually identify flat regions vs sharp peaks
  3. Sensitivity Analysis: Perturb each parameter individually, measure degradation. High-sensitivity params indicate fragile dimensions.

Why It Helps

Directly addresses Problem 1 (Parameter Fragility). The Atlas-ASX +/-15% perturbation causing 76% CAGR degradation is textbook parameter island behavior. Selecting high plateau-score parameters ensures the entire neighborhood performs similarly.

Python Implementation

import numpy as np

def compute_plateau_score(params, param_ranges, backtest_fn,
                          radius_pct=0.15, n_samples=20, threshold_pct=0.50):
    base_metric = backtest_fn(params)
    if base_metric <= 0:
        return 0.0
    rng = np.random.RandomState(42)
    neighbor_metrics = []
    for _ in range(n_samples):
        perturbed = {}
        for key, val in params.items():
            lo, hi = param_ranges[key]
            factor = rng.uniform(1 - radius_pct, 1 + radius_pct)
            new_val = np.clip(val * factor, lo, hi)
            if isinstance(val, int): new_val = int(round(new_val))
            perturbed[key] = new_val
        neighbor_metrics.append(backtest_fn(perturbed))
    threshold = base_metric * threshold_pct
    stable_frac = sum(1 for m in neighbor_metrics if m >= threshold) / n_samples
    mean_ratio = np.mean(neighbor_metrics) / base_metric
    return stable_frac * mean_ratio

1.1 Parameter Plateau Detection (PSO-Based)

CRITICAL Medium Complexity

What

Parameter plateau detection identifies broad, stable regions in parameter space where performance remains consistent under small perturbations, as opposed to sharp "parameter islands" that indicate overfitting. Wu et al. (2024) developed a plateau score algorithm that quantifies the extent of stability around each parameter combination, combined with Particle Swarm Optimization (PSO) for efficient search in high-dimensional spaces.

A parameter plateau is defined as a continuous block of parameter combinations with performance comparable to nearby ranges. The plateau score measures how robust a parameter set is — higher scores indicate broader, more stable performance regions. Traditional brute-force optimization finds the single best point (often a sharp peak/island); plateau-seeking optimization finds the best region.

Three approaches to finding plateaus exist in the literature:

  1. PSO with Plateau Score (Wu et al. 2024): Use PSO to explore parameter space; for each candidate, evaluate neighbors in a defined radius and compute a plateau score = fraction of neighbors meeting performance threshold × mean neighbor performance / base performance
  2. Heatmap Visualization: For low-dimensional spaces (2-3 params), plot performance heatmaps and visually identify flat regions vs sharp peaks. Robot Wealth recommends this as a first diagnostic step.
  3. Sensitivity Analysis: Perturb each parameter individually and measure performance degradation. Parameters with high sensitivity indicate fragile dimensions (already partially implemented in Atlas-ASX param_stability_report.py).

Why It Helps

Directly addresses Problem 1 (Parameter Fragility). The Atlas-ASX system's ±15% perturbation causing 76% CAGR degradation is the textbook symptom of sitting on a parameter island. By selecting parameters with high plateau scores instead of maximum backtest performance, the system would naturally become robust to perturbation because the entire neighborhood of the selected parameters performs similarly.

                          radius_pct=0.15, n_samples=20, threshold_pct=0.50):

    Higher score = broader plateau = more robust parameters.
    """
    base_metric = backtest_fn(params)  # e.g., CAGR or composite
    if base_metric <= 0:
        return 0.0

    rng = np.random.RandomState(42)
    neighbor_metrics = []

    for _ in range(n_samples):
        perturbed = {}
        for key, val in params.items():
            lo, hi = param_ranges[key]
            factor = rng.uniform(1 - radius_pct, 1 + radius_pct)
            new_val = np.clip(val * factor, lo, hi)
            if isinstance(val, int):
                new_val = int(round(new_val))
            perturbed[key] = new_val
        neighbor_metrics.append(backtest_fn(perturbed))

    # Plateau score components:
    threshold = base_metric * threshold_pct
    stable_frac = sum(1 for m in neighbor_metrics if m >= threshold) / n_samples
    mean_ratio = np.mean(neighbor_metrics) / base_metric
    plateau_score = stable_frac * mean_ratio
    return plateau_score

1.2 Objective Function Smoothing / Gaussian Averaging

CRITICAL Low Complexity

What

Instead of optimizing the raw objective function (CAGR, Sharpe, etc.) at each parameter point, smooth the objective by averaging over a Gaussian-weighted neighborhood around each point. This is fundamentally different from the Atlas-ASX v9.3 approach (which averaged the final parameters between v9.1 and v9.2) — here we average the objective function values across nearby parameter values before optimization.

The smoothed objective at parameter vector θ is:

f̂(θ) = Σi wi · f(θ + εi) / Σi wi
where wi = exp(-||εi||² / 2σ²) and εi are perturbation vectors

This creates a "landscape" where sharp peaks are flattened and broad plateaus are preserved. Optimization on the smoothed landscape naturally selects plateau centers.

Why It Helps

Directly addresses Problem 1 (Parameter Fragility) and Problem 2 (IS/OOS gap). Sharp peaks in the raw objective correspond to overfitting; smoothing removes them. The Atlas-ASX v9.3 blend failed because parameter averaging doesn't smooth the landscape — it just picks an arbitrary midpoint between two peaks.
