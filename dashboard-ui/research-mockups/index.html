Atlas Dashboard / Design Exploration

Research tab rewrite — 4 design variants

Four parallel prototypes of the proposed knowledge-layer dashboard, each taking a different strong opinion about how to surface the same data. All four consume an identical mock dataset (30 days of synthetic sources, claims, contradictions, lifecycle events, and discovery funnel data) so the comparison is about design, not data.

Shared mock data: shared/mock-data.json Click any card to open the prototype
Variant A

Triage Inbox

Contradictions are like email. Your daily job is to clear the queue.

Three-pane layout (filters / list / detail drawer), keyboard-first (j/k navigate, r resolve, / search), inline resolve dialog, right-side detail with paper abstract + claim-vs-measured cards. Strategies / sources / lifecycle live as secondary tabs above the list.

Best if you live in this tab every morning clearing contradictions and want it to feel like Linear or Superhuman.

Three-pane Keyboard-driven Workflow-first
Variant B

Newsroom

Tell me the story. Big headlines, prose, not tables.

Magazine layout. Hero "Today's top story" reads like a journalist wrote it (the #1 contradiction explained in a paragraph). Front-page section cards: Lifecycle wire, Strategy spotlight, Discovery digest. Serif headlines, generous whitespace, no charts. Tables only in the appendix at the bottom.

Best if you want to skim "what changed today" the way you'd read a Bloomberg morning briefing, and you trust the system to write decent prose for you.

Editorial Prose-led Chart-free
Variant C

Mission Control

Ops console. Glance at the page and see everything at once.

Dense 10-panel grid above the fold. System-health strip, severity counters (big numerals), lifecycle distribution, discovery funnel, top contradictions, transition feed, queue health, extraction confidence, digest sparkline, strategy tiles. Monospace everywhere, color-coded status dots, no animations, auto-refresh indicator.

Best if you want this tab to feel like Bloomberg / k9s / a NASA mission-control screen, and you check it as a health monitor multiple times per day.

10 panels Dense Monospace
Variant D

Funnel Analytics

Discovery is a pipeline. Show me the funnel; let me drill in.

Horizontal funnel with 5 stages (papers found → filtered → specs → strategies → QC) sized by 7-day volume, drop-off % between each. Stacked time-series below. Contradictions over time + resolution velocity side-by-side. Dynamic diagnostics callouts ("filter pass-rate dropped this week", "one day with 0 specs — likely rate-limit") computed from real data.

Best if you treat this as an analytics dashboard answering "where is the system bottlenecked?" — Amplitude / Mixpanel style.

Funnel-first Chart-heavy Diagnostics

All four prototypes are self-contained HTML with embedded CSS/JS and a shared mock dataset. No build step, no backend, no React. They are pure design references — pick the direction that resonates, and the chosen variant gets reimplemented as React components against the real /api/knowledge/* endpoints.

Try opening two in side-by-side browser windows to compare directly. Don't pick on aesthetics alone — pick on which information architecture makes your daily workflow faster.
